# نظام التصميم - منصة المساعدات الإنسانية

## مبادئ التصميم الأساسية

### 1. الوضوح والبساطة
- **الوضوح أولاً**: كل عنصر يجب أن يكون واضح الغرض والوظيفة
- **البساطة في التفاعل**: تقليل عدد الخطوات المطلوبة لإنجاز المهام
- **التركيز على المحتوى**: التصميم يدعم المحتوى وليس العكس

### 2. الاتساق والتوحيد
- **مكونات موحدة**: استخدام نفس المكونات عبر النظام
- **ألوان متسقة**: نظام ألوان موحد لجميع الحالات
- **تباعد منتظم**: استخدام نظام تباعد ثابت

### 3. إمكانية الوصول
- **تباين ألوان كافي**: ضمان قراءة واضحة لجميع المستخدمين
- **أحجام خطوط مناسبة**: سهولة القراءة على جميع الأجهزة
- **دعم لوحة المفاتيح**: إمكانية التنقل بالكيبورد

## أنماط التخطيط (Layout Patterns)

### 1. الشريط الجانبي
- **العرض**: 256px (w-64)
- **الخلفية**: أبيض مع حدود رفيعة
- **العناصر النشطة**: خلفية فاتحة مع مؤشر جانبي

### 2. المحتوى الرئيسي
- **الحشو**: 32px من جميع الجهات
- **الحد الأقصى للعرض**: لا يوجد (full width)
- **التباعد بين الأقسام**: 32px

### 3. الجداول
- **رأس الجدول**: خلفية رمادية فاتحة (bg-gray-50)
- **الصفوف**: تأثير تحويم خفيف
- **الحدود**: رفيعة ورمادية فاتحة

## الأيقونات

### مكتبة الأيقونات
- **المصدر**: Lucide React
- **الحجم القياسي**: 20px (w-5 h-5)
- **الحجم الصغير**: 16px (w-4 h-4)
- **الحجم الكبير**: 24px (w-6 h-6)

### استخدام الأيقونات
- **مع النصوص**: تباعد 8px بين الأيقونة والنص
- **في الأزرار**: أيقونة + نص مع تباعد مناسب
- **في البطاقات**: أيقونات ملونة في مربعات صغيرة

## إرشادات الاستخدام

### متى تستخدم كل لون
- **الأزرق**: الإجراءات الأساسية، الروابط، المعلومات
- **الأخضر**: النجاح، الموافقة، الإضافة
- **الأحمر**: الأخطاء، الحذف، التحذيرات الحرجة
- **البرتقالي**: التحذيرات، الانتظار، المراجعة
- **البنفسجي**: الميزات الخاصة، الإعدادات المتقدمة

### أولويات الأزرار
1. **زر أساسي واحد فقط** في كل شاشة
2. **أزرار ثانوية** للإجراءات الأقل أهمية
3. **أزرار النص** للإجراءات الاختيارية

## الاستجابة (Responsive Design)

### نقاط التوقف (Breakpoints)
```css
/* الهاتف المحمول */
@media (max-width: 768px) { ... }

/* الجهاز اللوحي */
@media (min-width: 768px) and (max-width: 1024px) { ... }

/* سطح المكتب */
@media (min-width: 1024px) { ... }
```

### قواعد الاستجابة
- **الشريط الجانبي**: يتحول لقائمة منسدلة على الهاتف
- **الجداول**: تصبح قابلة للتمرير أفقياً
- **البطاقات**: تتكيف مع عرض الشاشة

## إمكانية الوصول (Accessibility)

### تباين الألوان
- **الحد الأدنى**: 4.5:1 للنصوص العادية
- **النصوص الكبيرة**: 3:1 للعناوين والنصوص الكبيرة
- **العناصر التفاعلية**: 3:1 للأزرار والروابط

### دعم لوحة المفاتيح
- **Tab**: التنقل بين العناصر
- **Enter/Space**: تفعيل الأزرار والروابط
- **Escape**: إغلاق النوافذ المنبثقة

### النصوص البديلة
- **الأيقونات**: نصوص بديلة واضحة
- **الصور**: وصف مناسب للمحتوى
- **الأزرار**: تسميات واضحة للغرض

## قواعد التطوير

### 1. استخدام Tailwind CSS
- **الأولوية للفئات الجاهزة**: استخدم فئات Tailwind بدلاً من CSS مخصص
- **التسمية المتسقة**: استخدم نفس الفئات للعناصر المتشابهة
- **تجنب القيم المخصصة**: التزم بنظام التباعد والألوان المحدد

### 2. هيكلة المكونات
- **مكون واحد لكل ملف**: كل مكون في ملف منفصل
- **استيراد منظم**: ترتيب الاستيرادات (React، المكتبات، المكونات المحلية)
- **TypeScript**: استخدام الأنواع لجميع المكونات

### 3. إدارة الحالة
- **useState للحالة المحلية**: للبيانات الخاصة بالمكون
- **Props للتواصل**: تمرير البيانات بين المكونات
- **Context للحالة العامة**: للبيانات المشتركة

## إرشادات الصيانة

### 1. تحديث المكونات
- **اختبار التغييرات**: تأكد من عمل المكون في جميع الحالات
- **توثيق التغييرات**: تحديث التوثيق عند إضافة ميزات جديدة
- **مراجعة الاتساق**: التأكد من توافق التغييرات مع النظام العام

### 2. إضافة مكونات جديدة
- **اتباع النمط الموجود**: استخدام نفس الأنماط والألوان
- **اختبار إمكانية الوصول**: التأكد من دعم جميع المستخدمين
- **توثيق الاستخدام**: إضافة أمثلة وإرشادات للاستخدام
